**Learning project (2022):** Exploring Web Workers for parallel operations.

**Concept:** Simulated concurrent user purchases using Web Workers to offload processing from the main thread. Each worker handled an independent purchase flow, demonstrating:
- Worker lifecycle and messaging
- Shared state challenges
- Node.js backend integration

**What I learned:** Concurrency adds complexity—synchronization, race conditions, and state management become critical. Modern approach would use SharedWorkers or structured state management.

**Status:** Historical project showing early exploration of parallel JavaScript.
